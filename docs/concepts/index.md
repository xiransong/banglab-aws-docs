# Concepts

AWS is a large and complex platform.
This section introduces **only the concepts needed to use AWS effectively in BangLab**.

The goal here is to provide a **simple mental model** so that:

- the login flow makes sense,
- access restrictions feel intentional,
- and later tutorials are easier to follow.

---

## What This Section Covers

In this section, you will learn:

- **Accounts and Access**  
  How AWS separates *accounts*, *users*, and *permission sets*, and how access is granted via SSO.

- **Organization and Billing**  
  How BangLab uses an AWS Organization, where credits live, and why users do not manage billing.

- **EC2 Basics**  
  What EC2 is, how GPU instances fit in, which region we use, and how vCPU quotas work.

- **S3 Basics**  
  What S3 is (and is not), how data is organized, and why it behaves differently from a filesystem.

---

## What This Section Does *Not* Cover

This section intentionally does **not** cover:

- how to log in
- how to configure the AWS CLI
- how to launch instances
- how to upload data

Those topics are covered later in:

- **Access & Capabilities**
- **Onboarding**
- **EC2**
- **S3**
